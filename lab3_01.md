## 二、背景知识

### 2.1 什么是 API

API（Application Programming Interface，应用程序编程接口）是软件之间"对话"的约定方式。你可以把它理解为一个"服务窗口"：

```
你的程序  ──发送请求──→  API 服务（云端）
         ←─返回数据──   天气数据 / 地图数据 / AI 回复
```

在日常生活中，你使用的几乎所有 App 都在背后调用各种 API：
- 打开天气 App → 调用天气 API 获取实时天气
- 使用高德/百度地图 → 调用地图 API 渲染地图和规划路线
- 和 AI 聊天 → 调用大模型 API 获取回复

### 2.2 API Key：你的"数字钥匙"

大多数 API 需要一个 **API Key** 来识别你的身份并控制调用量。它就像你的身份证：
- 每个开发者有自己的 Key
- API 服务商通过 Key 统计你的使用量
- 大部分 API 对个人开发者提供**免费额度**，足够学习和小型项目使用

> **安全提醒：** API Key 是敏感信息，**不要**将其直接写在公开的代码中或上传到 GitHub。

### 2.3 本次实验要做什么

你将构建一个**地图天气 AI 助手**，它组合了 3 个 API 的能力：

| 步骤 | 使用的 API | 做什么 |
|------|-----------|--------|
| ① 地图选点 | 高德地图 JS API | 在交互式地图上点击选择一个地点，获取地名和经纬度 |
| ② 获取天气 | 和风天气 API（或高德天气 API） | 根据经纬度获取该地点的实时天气和预报 |
| ③ AI 对话 | 大模型 API（智谱/DeepSeek/豆包等） | 把位置和天气信息作为上下文，与 AI 进行智能对话 |

最终效果：用户在地图上选一个地方，页面自动显示当地天气，然后可以问 AI 类似"今天适合去这里户外活动吗""需要带伞吗""推荐附近什么活动"等问题。

## 四、实验任务

### 核心任务：构建"地图天气 AI 助手"

**技术栈：** HTML + CSS + JavaScript（纯前端）

**功能要求：**

| 模块 | 功能 |
|------|------|
| 地图交互 | 嵌入高德地图，用户可以在地图上点击选择地点，显示地名和经纬度 |
| 天气展示 | 根据选中地点的经纬度，自动调用天气 API 获取并展示当前天气信息（温度、天气状况、风速等） |
| AI 对话 | 在页面中提供聊天区域，用户可以基于当前选中地点和天气，与 AI 进行对话（如"今天适合户外活动吗""需要带什么衣服"） |

**核心要点：**
- AI 对话时，需要将**当前位置信息**和**天气数据**作为上下文注入给大模型（通过 System Prompt 或 User Message），这样 AI 才能给出有针对性的建议
- 3 个 API 的数据要形成**串联**：地图选点 → 天气查询 → AI 对话上下文

### 开发方式

- **使用 Trae 协作完成**全部开发，体验"用 AI 构建多 API 集成应用"
- **多轮对话开发**：建议分步完成，每轮聚焦一个子目标：
  1. 第一轮：先在页面中嵌入高德地图，实现点击获取经纬度
  2. 第二轮：接入天气 API，点击地图后自动显示天气
  3. 第三轮：接入大模型 API，实现带天气上下文的 AI 对话
  4. 后续轮次：优化 UI、修复 bug、添加更多功能

### 自主拓展（可选，鼓励探索）

在完成核心功能后，可以自主选择以下方向进行拓展：

- **多日预报展示**：展示未来 3-7 天的天气预报，AI 可以基于多日天气给出出行规划
- **UI 优化**：优化聊天界面（消息气泡、打字效果）、天气信息的可视化展示（天气图标、温度曲线）
- **位置搜索**：添加搜索框，支持输入地名搜索跳转到对应位置（使用高德地理编码 API）
- **AI 流式输出**：让 AI 的回复逐字显示（streaming），提升交互体验
- **历史记录**：保存用户查询过的地点和对话记录（localStorage）
- **其他你感兴趣的功能**


### 8.2 关于高德地图 JS API

- 高德地图 JS API 2.0 需要通过特定方式加载，通常使用官方的 `AMapLoader`
- 地图上的点击事件可以获取到经纬度，再通过**逆地理编码（regeo）**接口将经纬度转换为地名
- 具体的加载方式和事件绑定，可以直接让 Trae 帮你实现——告诉它你要"在网页中嵌入高德地图，点击地图获取地名和经纬度"

### 8.3 关于天气 API 的数据处理

如果使用和风天气：
- 返回数据本身就是中文，字段很直观（如 `text: "晴"`, `temp: "28"`, `windDir: "东南风"`）
- 可以直接用于页面展示和 AI 对话上下文

如果使用高德天气 API：
- 需要先通过逆地理编码获取 `adcode`（城市编码），再用 adcode 查天气
- 返回数据也是中文（如 `weather: "晴"`, `temperature: "28"`）
- 好处是不用额外注册，和地图用同一个 Key

### 8.4 关于大模型 API 的调用

- 在纯前端中，可以直接使用 `fetch` 调用大模型 API（大部分平台支持 CORS）
- 关键是设计好 System Prompt 或在 User Message 中注入天气上下文，例如：

```
用户当前选择的位置：上海市浦东新区
当前天气：晴，温度 28°C，体感温度 31°C，相对湿度 65%，东南风 3 级
未来 3 天：明天多云转阴 25-30°C，后天小雨 22-27°C

用户问题：今天适合户外活动吗？
```

- 这样 AI 就能基于具体的天气数据给出有针对性的建议

### 8.5 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 地图加载不出来 | Key 配置错误或安全密钥未配置 | 检查 Key 和 jscode，确认 localhost 在白名单中 |
| 天气 API 请求失败 | 跨域问题、参数错误或 Key 未配置 | 和风天气注意使用 HTTPS 和正确的 Key；高德天气注意 adcode 格式 |
| 大模型 API 报错 | Key 无效或 base_url 错误 | 检查平台文档，确认 Key 和 URL |
| AI 回复和天气无关 | 天气数据未注入到对话上下文 | 检查 messages 中是否包含天气信息 |
